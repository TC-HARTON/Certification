## 全体観

HARTON Stella の最重要 KPI は、最終的に **「HARTON 本体（tcharton.com）への送客と契約」**に直結する。
サブドメイン自体は無料運営だが、ブランド資産として HARTON のサービス需要を生み出す装置として設計する。

---

## 階層構造

```
[最上位 KPI（事業成果）]
  ↓
HARTON サービス契約数 (月 X 件)
  ↑（送客）
  ↓
[中間 KPI（送客）]
tcharton.com への質の高いリード流入数
  ↑（誘導）
  ↓
[基盤 KPI（コンテンツ価値）]
PV / セッション / 滞在時間 / 被リンク
```

```
■ 数値サマリー
  - 総 PV / セッション / ユニーク

  - ★区分の変動（上昇 / 下降 件数）
  - 新規認定 / 認定取消 件数

■ コンテンツパフォーマンス
  - 最も見られた業種ページ TOP 5
  - 最も見られた個別事業者ページ TOP 5
  - 最もクリックされた公式サイト TOP 5

■ 改善アクション
  - 直帰率が高いページ → 要改善
  - 滞在時間が短いページ → 要改善

■ 次月予定
  - 新規認定予定 件数
  - メディア対応予定
  - 改訂アクション
```

## 計測ツール構成

| ツール | 用途 |
|---|---|
| Google Analytics 4 | PV / セッション / 滞在時間 / 流入経路 / イベント |
| Google Search Console | 検索クエリ / 表示順位 / 被リンク |
| Cloudflare Web Analytics | サーバーサイドアクセス（GA4 補完）|
| ahrefs / semrush | バックリンク詳細・競合比較（Phase 1 以降）|
| HARTON 自社 CRM | サービス契約者の流入経路追跡 |
| 月次手動集計 | メディア掲載・取材依頼・特殊指標 |

### UTM パラメータ設計

tcharton.com への送客時には UTM 付与：

```
?utm_source=stella.tcharton.com
&utm_medium=referral
&utm_campaign=apply_page  # or other page
&utm_content=cta_button   # or other element
```

→ tcharton.com 側 GA4 で「HARTON Stella 経由」を正確に分離計測。

---

## 計測の落とし穴と対応

| 落とし穴 | 対応 |
|---|---|
| GA4 で外部リンククリックは標準計測されない | カスタムイベント `outbound_link_click` を実装 |
| バッジ埋込み率はバックリンク調査でしか計測不可 | 月次で ahrefs / semrush を確認、または自社サイトに「埋込みコード使用通知」機能 |
| 「契約が HARTON Stella 経由か」の特定 | 申込時に「どこで知りましたか?」項目必須 + UTM |
| 直帰率の解釈 | 業種ページから個別事業者ページへの流れは「直帰でない」と設計（イベント発火）|
