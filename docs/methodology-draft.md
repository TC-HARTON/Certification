## メインページ `/methodology/` の本文草案

### Section 1: ヒーロー（159 字）

```
公正に測る、ということ。

数値だけでも、感情だけでも測れない領域がある。
HARTON Stella は、移住者の客観性と
15 年のエンジニア経験を礎に、
2,554 項目の機械検証で地域のサイトを評価する。

評価方法のすべてを、ここに公開する。
情にも、規模にも、流行にも左右されない、
4 軸の検証基準を。
```

### Section 2: 評価の 5 原則

#### 2.1 透明性
> 評価項目・閾値・実施手順を全公開する。スコア算出ロジックも公開する。
>
> ブラックボックスの認定は、認定の意味を持たない。
> あなたが今ご覧いただいているこのページが、HARTON Stella の核である。

#### 2.2 客観性
> 主観的判断を排除し、すべて scanner（独自開発の機械検証）による自動計測のみで評価する。
>
> 人間の好みや感情、業種・規模への偏見を入れない。
> 数値だけが語る。これが移住者である私の選んだ道だ。

#### 2.3 ポジティブセレクション
> 「悪いサイトを晒す」のではなく「良いサイトを称える」。
> 掲載基準（★3 以上）に達しないサイトは、本サイトに掲載しない。
>
> ミシュランガイドに低評価レストランが載らないのと同じ。
> 認定とは、本来そういうものだ。

#### 2.4 公平性
> 業種・規模・HARTON との取引関係に**一切依存しない**。
> スコアは機械検証の結果のみで決定する。
>
> HARTON が制作したサイトであろうと、競合制作会社のサイトであろうと、
> 同じ scanner、同じ基準、同じスコア。

#### 2.5 進化
> 年次で基準を改訂する。WEB の進化に追従し、
> 過去の評価も改訂時にアーカイブとして残す。
>
> 2026 年に ★4 を獲得したサイトは、2027 年の新基準で再評価される。
> 評価が変わっても、過去の名誉は記録として永続する。

### Section 3: 4 軸スコアリング概観

```
HARTON Stella のスコアは、4 つの軸で構成される。
各軸は scanner（独自開発のスキャナー）が機械検証する。
```

#### 軸の概要表

| 軸 | 軸名 | 何を測るか | 詳細ページ |
|---|---|---|---|
| **A** | 基礎・身だしなみ | サイトが訪問者を受け入れる準備（SSL・速度・基本構造）| `/methodology/technical/` |
| **B** | 防御力・生存率 | サイトが攻撃やトラブルから自分を守れるか（脆弱性・WAF・CSP）| `/methodology/security/` |
| **C** | AI 検索適応 | AI 検索エンジンが正しくサイトを理解できるか（JSON-LD・セマンティック）| `/methodology/ai-search/` |
| **D** | 経営インパクト | サイトが事業者の経営に貢献できているか（OGP・モバイル・A11y）| `/methodology/business-impact/` |

⚠️ **TBD**: 各軸の重み（25% × 4 か別の比率か）は scanner/スキャンプロンプト.txt から verbatim 確定が必要（§0.0.9）。

#### 視覚化案

レーダーチャート（4 軸）の架空サンプルを 1 つ表示し、「このような形で各事業者を評価する」と説明。

### Section 4: S クラス必須 5 条件

```
★5（HARTON S-Class Certified）を冠する事業者は、
以下 5 条件をすべて満たす：
```

| # | 条件 | 出典 |
|---|---|---|

### Section 5: ★区分の閾値

```
HARTON Stella は 5 段階★で評価する。
掲載対象は ★3 以上のみ（ポジティブセレクション）。
```

#### ★区分表（暫定）

| ★ | 呼称 | 4 軸合計 | 致命的 NG | S 条件 | 主要要件 |
|---|---|---|---|---|---|
| ★★★ | HARTON Stella | TBD | ゼロ必須 | 不問 | SSL / 基本構造 / モバイル対応 |
| ★★★★ | HARTON 優良 | TBD | ゼロ必須 | 4/5 | + LCP < 2.5s / WCAG 2.2 A |
| ★★★★★ | HARTON S-Class | TBD | ゼロ必須 | 5/5 | + LCP < 1.2s / WCAG 2.2 AA / SPEC 全項目 |

⚠️ **TBD**: 70/85/95 等の閾値は Phase 0 のパイロット 30 件のスコア分布から逆算決定。

### Section 6: 評価サイクル

```
月次再スキャン     ← 全掲載事業者を毎月再評価
四半期レビュー     ← ★区分変動の傾向分析
年次基準改訂      ← 4 軸・S 条件の見直し
即時対応       ← 重大脆弱性発見時は即時★区分変更通知
```

#### スコア変動時の運用

| 変動 | 対応 |
|---|---|
| ★が上昇 | 当該事業者に通知、認定書更新、サイトに反映、note・SNS で告知 |
| ★が下降 | 当該事業者に通知、改善ガイダンス送付、3 ヶ月猶予後にサイト反映 |
| 致命的 NG 検出 | 即時通知、24 時間以内に掲載一時停止判断 |

### Section 7: データの取扱いと倫理

```
評価対象は「公開情報」のみ。
パスワード保護領域、個人情報、内部システムは
一切スキャンしない。
```

#### 倫理ガイドライン

| 項目 | 方針 |
|---|---|
| スキャン対象 | 公開 URL のみ。robots.txt の Disallow を尊重 |
| スクリーンショット | 批評目的の引用として最小限。商用転用禁止 |
| 掲載拒否権 | `/opt-out/` でいつでも除外可能。理由不要、即時対応 |
| 個人情報 | 法人情報・公開連絡先のみ収集。個人事業主は屋号レベルのみ |
| データ保管 | スキャン結果は HARTON ローカルのみ。第三者提供なし |
| 訂正請求 | 評価誤りの訂正請求を `/contact/` で受付 |
| 法令遵守 | GDPR / 個人情報保護法 2022 改正対応、削除請求に応じる |

## サブページの共通テンプレート

`/methodology/technical/` 等の各軸詳細ページは、以下の共通構造で：

```
[ヒーロー] 軸名 + 一行サマリー

1. なぜこの軸が重要か
   - 事業者目線
   - ユーザー目線
   - AI 検索目線

2. 評価項目リスト（10-30 項目）
   - 各項目の意味
   - 合格基準
   - 自分でチェックする方法

3. 改善のヒント
   - ★3 → ★4 → ★5 への昇格ガイダンス

4. 関連リソース
   - 公式ドキュメント・規格書へのリンク
```

→ 各軸の詳細項目は scanner/spec-checker.js / スキャンプロンプト.txt から verbatim 取得して反映。
