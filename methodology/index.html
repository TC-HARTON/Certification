HARTON Certified は WEB サイトを 4 つの観点で並列独立評価する。各観点は scanner.py で 45+ 項目を機械検証し、合計 2554 項目の SPEC v3.4 と整合する。出典: Google Web Vitals 公式 / W3C WCAG 2.2。

「公正に測るとは、評価方法のすべてを公開することだ」 — HARTON Certified 評価原則

2026-04-27 公開

評価方法

評価の 5 原則

  • 透明性: 評価項目・閾値・実施手順を全公開する。
  • 客観性: 主観的判断を排除し、scanner.py による自動計測のみ。
  • ポジティブセレクション: 良いサイトを称え、低評価サイトは公開しない。
  • 公平性: 業種・規模・HARTON との取引関係に一切依存しない。
  • 進化: 年次で基準を改訂し、過去評価もアーカイブとして保持する。

4 軸機械検証（並列独立評価）

HARTON Certified 4 軸の役割と詳細導線
軸名称評価対象詳細
A基礎・身だしなみSSL / 速度 / 基本構造A 軸詳細
B防御力・生存率脆弱性 / WAF / CSPB 軸詳細
CAI 検索適応JSON-LD / セマンティックC 軸詳細
D経営インパクトOGP / モバイル / A11yD 軸詳細

軸間の重み比率は採用しない。各軸は項目別の事実減点で評価し、4 軸の合算が総合スコアとなる（MASTER-PLAN §3.2）。

★区分の閾値（MASTER-PLAN §3.4）

★3（HARTON Certified）= 総合 70 点以上 + 致命的 NG 0 件。★4（HARTON 優良）= 総合 80 点以上 + S 条件 4/5。★5（HARTON S-Class）= 総合 90 点以上 + S 条件 5/5 + 致命的 NG 0 件。

致命的 NG（一発除外条件）

  • HTTPS 非対応 — 出典: IPA「安全なウェブサイトの作り方」
  • SSL 証明書エラー（期限切れ・無効）
  • WP 管理面露出（wp-login.php / readme.html / xmlrpc.php）
  • CMS バージョン情報露出（generator meta タグ等）

★区分問わず一発除外する。scanner.py Phase 9d で機械検出を実装済（2026-04-26）。

評価サイクル

  • 月次: 全掲載事業者を再スキャン、★区分変動を反映
  • 四半期: ★区分変動の傾向分析、改善ガイダンス更新
  • 年次: 4 軸・S 条件の見直し、SPEC v3.x との整合確認
  • 即時: 重大脆弱性発見時は 24 時間以内に掲載一時停止判断
